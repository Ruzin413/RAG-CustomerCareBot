RAG Chatbot API Reference
==========================

Base URL: http://localhost:8001/CustomerCare

Authorization:
--------------
All endpoints require a static API Key for security.
- Header: X-API-Key: ft-customer-care-secret-2026
- Cookie: admin_token=ft-customer-care-secret-2026
- Logic: The backend `verify_token` dependency checks both Header and Cookie. If either contains the correct token, access is granted.

1. Chat & Processing
--------------------
POST /chat
Description: Public chat endpoint. Processes user queries and returns a grounded AI response or navigation command.
- JSON Payload: { "message": String, "kb_name": String }
- Response: { 
    "status": "success", 
    "answer": String, 
    "intent": String, 
    "redirect_to": String, // included if intent is "navigate"
    "source": String,      // "Knowledge Base" or "Fallback"
    "metadata": {
        "timestamp": String,
        "timing_breakdown": { "total_latency": String }
    }
  }

POST /process
Description: Admin-only processing endpoint (requires token). Same logic as /chat.
- Headers: { "X-API-Key": "ft-customer-care-secret-2026" }
- JSON Payload: { "message": String, "kb_name": String }
- Response: Same as /chat

POST /clear-cache
Description: Manually unloads AI models from memory (RAM/VRAM). Useful for clearing resources without restarting the server.
- Response: { "status": "success", "message": String }

GET /health
Description: Basic health check for monitoring.
- Response: { "status": "healthy", "service": "Chatbot API", "timestamp": String }

2. Knowledge Base Management
----------------------------
POST /upload
Description: Initializes a new Knowledge Base system or adds files to an existing one.
- Headers: { "X-API-Key": "ft-customer-care-secret-2026" }
- Multipart Form: kb_name (String), file (Files), custom_names (JSON Array string)
- Response: { 
    "status": "success", 
    "total_chunks": Integer,
    "processed_files": List,
    "failed_files": List 
  }

POST /knowledge-bases/<kb_name>/append
Description: Adds more documents specifically to an existing Knowledge Base.
- Headers: { "X-API-Key": "ft-customer-care-secret-2026" }
- Multipart Form: file (Files), custom_names (JSON Array string)
- Response: { "status": "success", "total_chunks": Integer }

GET /knowledge-bases
Description: Returns a registry of all active Knowledge Systems.
- Headers: { "X-API-Key": "ft-customer-care-secret-2026" }
- Response: { "status": "success", "knowledge_bases": { "Name": { "jsonfile": String, "binfile": String } } }

POST /knowledge-bases
Description: Adds or updates a knowledge base configuration in the registry.
- Headers: { "X-API-Key": "ft-customer-care-secret-2026" }
- JSON Payload: { "name": String, "jsonfile": String, "binfile": String }
- Response: { "status": "success", "message": String }

DELETE /knowledge-bases/<name>
Description: Completely removes a Knowledge Base system and its files (.bin and .json).
- Headers: { "X-API-Key": "ft-customer-care-secret-2026" }
- Response: { "status": "success", "message": String }

GET /stats
Description: Returns the total number of chunks currently loaded in the active vector store.
- Response: { "status": "success", "total_chunks": Integer, "message": String }

3. Learning Loop (Admin)
------------------------
GET /chat-history
Description: Retrieves logs from the centralized `chat_history.json`.
- Headers: { "X-API-Key": "ft-customer-care-secret-2026" }
- Query Params: kb_name (Optional String), page (Int), page_size (Int)
- Response: { 
    "status": "success",
    "items": List, 
    "total": Int, 
    "page": Int, 
    "total_pages": Int 
  }

POST /unverified/update
Description: Edits and verifies a chat log entry. This moves the item into the "verified" category for RAG retrieval.
- Headers: { "X-API-Key": "ft-customer-care-secret-2026" }
- JSON Payload: { "chunk_id": String, "text": String, "kb_name": String (Optional) }
- Response: { "status": "success", "message": String }

POST /unverified/delete
Description: Permanently discards a chat interaction from the centralized history.
- Headers: { "X-API-Key": "ft-customer-care-secret-2026" }
- JSON Payload: { "chunk_id": String, "kb_name": String (Optional) }
- Response: { "status": "success", "message": String }

POST /memory/add
Description: Manually insert a pre-verified Q/A pair.
- Headers: { "X-API-Key": "ft-customer-care-secret-2026" }
- JSON Payload: { "question": String, "answer": String, "kb_name": String }
- Response: { "status": "success", "message": String }

4. Implementation Hooks (Frontend)
----------------------------------

const API_KEY = 'ft-customer-care-secret-2026';
const BASE_URL = 'http://localhost:8001/CustomerCare';

// 1. Chat Processing
const sendChat = async (message, kbName) => {
  const res = await fetch(`${BASE_URL}/process`, {
    method: 'POST',
    headers: { 
      'Content-Type': 'application/json', 
      'X-API-Key': API_KEY 
    },
    body: JSON.stringify({ message, kb_name: kbName })
  });
  const data = await res.json();

  // Navigation Hook:
  if (data.redirect_to) {
    console.log(`Navigating to: ${data.redirect_to}`);
    // window.location.href = data.redirect_to; // or use useNavigate() in React
  }

  return data;
};

// 2. Initial Upload / Create KB
const uploadKB = async (kbName, files) => {
  const formData = new FormData();
  formData.append('kb_name', kbName);
  files.forEach(f => formData.append('file', f));
  const res = await fetch(`${BASE_URL}/upload`, {
    method: 'POST',
    headers: { 'X-API-Key': API_KEY },
    body: formData
  });
  return await res.json();
};

// 3. Append to Existing KB
const appendDocs = async (kbName, files) => {
  const formData = new FormData();
  files.forEach(f => formData.append('file', f));
  const res = await fetch(`${BASE_URL}/knowledge-bases/${kbName}/append`, {
    method: 'POST',
    headers: { 'X-API-Key': API_KEY },
    body: formData
  });
  return await res.json();
};

// 4. List Knowledge Bases
const getKBs = async () => {
  const res = await fetch(`${BASE_URL}/knowledge-bases`, {
    headers: { 'X-API-Key': API_KEY }
  });
  return await res.json();
};

// 5. Delete Knowledge Base
const deleteKB = async (kbName) => {
  const res = await fetch(`${BASE_URL}/knowledge-bases/${kbName}`, {
    method: 'DELETE',
    headers: { 'X-API-Key': API_KEY }
  });
  return await res.json();
};

// 6. Get Chat History
const getHistory = async (kbName, page = 1) => {
  const url = kbName ? `${BASE_URL}/chat-history?kb_name=${kbName}&page=${page}` : `${BASE_URL}/chat-history?page=${page}`;
  const res = await fetch(url, {
    headers: { 'X-API-Key': API_KEY }
  });
  return await res.json();
};

// 7. Verify History Item
const verifyEntry = async (chunkId, newAnswer, kbName) => {
  const res = await fetch(`${BASE_URL}/unverified/update`, {
    method: 'POST',
    headers: { 
      'Content-Type': 'application/json', 
      'X-API-Key': API_KEY 
    },
    body: JSON.stringify({ chunk_id: chunkId, text: newAnswer, kb_name: kbName })
  });
  return await res.json();
};

// 8. Discard ChatHistory Item
const deleteHistoryItem = async (chunkId, kbName) => {
  const res = await fetch(`${BASE_URL}/unverified/delete`, {
    method: 'POST',
    headers: { 
      'Content-Type': 'application/json', 
      'X-API-Key': API_KEY 
    },
    body: JSON.stringify({ chunk_id: chunkId, kb_name: kbName })
  });
  return await res.json();
};

// 9. Manual Add Memory
const addManualMemory = async (question, answer, kbName) => {
  const res = await fetch(`${BASE_URL}/memory/add`, {
    method: 'POST',
    headers: { 
      'Content-Type': 'application/json', 
      'X-API-Key': API_KEY 
    },
    body: JSON.stringify({ question, answer, kb_name: kbName })
  });
  return await res.json();
};

// 10. Add/Update KB Config
const updateKBConfig = async (name, jsonfile, binfile) => {
  const res = await fetch(`${BASE_URL}/knowledge-bases`, {
    method: 'POST',
    headers: { 
      'Content-Type': 'application/json', 
      'X-API-Key': API_KEY 
    },
    body: JSON.stringify({ name, jsonfile, binfile })
  });
  return await res.json();
};

5. Navigation Configuration (Backend Logic)
------------------------------------------
The system uses a centralized map in `ai-services/navigation_utils.py` to handle redirects.

NAV_MAP = {
    "report": {
        "keywords": ["report"],
        "path": "/report"
    },
    "history": {
        "keywords": ["history"],
        "path": "/history"
    },
    "payment": {
        "keywords": ["payment"],
        "path": "/payment"
    }
}

To add a new redirect:
1. Open `ai-services/navigation_utils.py`.
2. Add a new entry to `NAV_MAP`.
3. The `HybridPipeline` will automatically detect the intent and return the `redirect_to` path.

6. Security & Token Logic (Backend)
----------------------------------
The system enforces security via the `verify_token` dependency in `app.py`:

ADMIN_TOKEN = os.getenv("ADMIN_TOKEN", "ft-customer-care-secret-2026")

async def verify_token(
    x_api_key: Optional[str] = Header(None), 
    admin_token: Optional[str] = Cookie(None)
):
    """Dependency to verify static token from header or cookie."""
    if x_api_key == ADMIN_TOKEN or admin_token == ADMIN_TOKEN:
        return True
    raise HTTPException(status_code=401, detail="Unauthorized: Invalid or missing API Token")
